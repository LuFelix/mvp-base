🚀 COMO USAR (3 FORMAS):


  ⭐ OPÇÃO 1 - Script Automatizado (RECOMENDADO):

     Da pasta raiz do projeto:
       $ ./backend/seed-docker.sh

     Da pasta do backend:
       $ ./seed-docker.sh

     Vantagens:
       ✓ Tudo automatizado
       ✓ Detecção automática de localização
       ✓ Mostra status dos containers
       ✓ Pausa antes de sair (você consegue ler tudo)


  📦 OPÇÃO 2 - Comando Direto (Docker):

     $ docker-compose exec backend npm run seed:verbose
     ou
     $ docker-compose exec -it backend npm run seed:verbose

     Rápido, mas sem o "script helper"


  🎯 OPÇÃO 3 - Entrar no Container (Manual):

     $ docker-compose exec -it backend bash
     $ npm run seed:verbose
     $ exit

✨ O QUE O NOVO SCRIPT (seed-docker.sh) FAZ:


  1️⃣  Detecta automaticamente sua localização
      └─ Funciona de qualquer pasta do projeto

  2️⃣  Valida docker-compose
      └─ Verifica se está instalado

  3️⃣  Valida containers
      └─ Mostra status de backend, frontend, db
      └─ Pausa se algum não estiver rodando

  4️⃣  Conecta ao container backend
      └─ docker-compose exec backend ...

  5️⃣  Executa o seed
      └─ npm run seed:verbose (COM TODOS OS LOGS!)

  6️⃣  Resumo visual
      └─ Mostra se foi sucesso ou erro

  7️⃣  PAUSA para confirmação
      └─ "Pressione ENTER para confirmar e sair..."
      └─ Você consegue ler tudo no terminal
      └─ Depois aperta ENTER e sai

  8️⃣  Próximos passos
      └─ Mostra as credenciais do admin
